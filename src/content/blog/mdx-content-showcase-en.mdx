## What this article is

This is a **buildable** MDX example, not a screenshot or pseudocode. It keeps normal writing in Markdown while using Astro components for information structures that are worth reusing.

> [!TIP]
> Start ordinary posts as `.md`. Choose `.mdx` only when you need article components like these, native HTML, or computed data.

> The post layout already supplies reading time, a table of contents, tags, related posts, language switching, views, and comments. Authors do not repeat those concerns in the body.

## Markdown remains the default

You can use **bold text**, *emphasis*, `inline code`, [external links](https://astro.build/), quotes, horizontal rules, and normal lists:

- One point
- A second point
  1. Nested steps work too
  2. They are useful for conditions or sequences

- [x] Make the title and description clear
- [x] Mark code blocks with a language
- [ ] Review the post in a real browser before publishing

## What this does not include

This showcase does not pretend that everything is already supported. Mermaid, KaTeX / MathJax, video players, PDF / Bilibili embeds, and ECharts / Chart.js are not wired up yet.

When one of them becomes a real writing need, first add a centrally maintained, accessible component and verify it in a browser. Future posts can then import that component without bringing their own temporary implementation.
