这几天各个论坛都在刷封号，有正价 Pro，有 Plus，有免费号，连用了三年的老号都没幸免。我把社区里被封的案例翻了个遍，发现规律其实挺清楚——而且大部分都能提前避开。

先说个反常识的结论：**这波封号，跟你拿 ChatGPT 聊了什么基本没关系。**很多人“啥也没干”就被杀，问题不在内容，在你这个号的“画像”——注册地、支付方式、登录 IP、有没有被多人共用。OpenAI 的风控盯的是这些，不是你的聊天记录。

所以“我没违规啊”这种委屈没用。风控不是看你违没违规，是看你这个号**像不像一个会被批量薅、批量倒卖、批量滥用的号**。像，就先杀了再说。

## 被封的号，扒出来这几个共性

翻了一圈案例，按中招概率从高到低排一下：

| 情况 | 风险 | 为什么被盯上 |
| --- | --- | --- |
| 一个号多人拼车 / 共享 | 最高 | 同一个账号被好几个人、好几个地方、好几个 IP 轮流登。在风控眼里这就是“账号被倒卖/批量操作”的标准画像。有人正价美区号分享给多人用，照样被杀。 |
| 用第三方 Agent 连 ChatGPT 账号（OAuth 授权） | 最高 | Hermes Agent、OpenClaw 这类第三方壳子，拿你的 ChatGPT OAuth 去驱动 Codex 或跑自动化。X 上这两天一批“零违规”被秒封的，几乎都是刚把账号连上这类工具就收到封号邮件。官方那头看到的，是你的账号正被一个非官方客户端操控，跟账号被盗用一个性质。 |
| 低价区账号（土区/菲区等）+ 礼品卡跨区充值 | 高 | 注册区域是便宜区，实际人在另一个地方用，支付又是礼品卡/跨区。区域对不上、支付来路不明，是这轮清洗的重点对象。 |
| 登录 IP 区域乱跳 / 频繁换节点 | 高 | 账号显示在土耳其，你的 IP 一会儿日本一会儿美国一会儿新加坡。正常人不会几小时横跨几个国家，这种漂移很扎眼。 |
| 机房 IP / 脏代理登录 | 中高 | 出口落在 AWS、各种 IDC 机房，或者一个被无数人薅过的共享代理。本身信誉就低，叠加上面几条直接放大。 |
| Codex 重度自动化使用 | 中 | 短时间打出海量 token、或者用非官方工具套壳驱动，容易被反滥用模型盯上。社区不少人怀疑这轮和 Codex 风控调整有关。 |
| 被人举报 | 中 | 有传言这轮和前两天有人集中举报有关。举报会让某一类号（尤其低价区）进入重点核查。 |

> 补一个这两天 X 上最扎眼的：一批“我啥违规都没有”被秒封的，共同点是把账号接进了 **Hermes Agent、OpenClaw 这类第三方 harness**——用你的 ChatGPT OAuth 去跑 Codex。这等于把账号交给一个官方不认识的客户端操控，刚连上就被封的案例一抓一大把。想用 Codex，就用官方 CLI 或官方 IDE 插件，别图省事套第三方壳。

反过来，社区里**没被封的**也有共性：一个号自己一个人用、登录 IP 固定干净、区域对得上的，基本没事；走中转 / 反代 API 的（不碰网页账号登录这条线）也大多平安。

> 把上面六条再压缩一下，你会发现**至少四条的根都在“登录这个号的 IP”上**：多人共享=多个杂 IP 登同一个号；区域乱跳=IP 和账号注册地对不上；机房脏 IP=IP 本身质量差。说白了，**账号的“画像干不干净”，一大半是由你用什么 IP 登决定的。**

## 为什么 IP 质量是这件事的命门

你可以这么理解 OpenAI 的判定逻辑：它给每个账号维护一张“行为画像”，而登录 IP 是画像里权重最高的几个信号之一。一个号如果长期从同一个干净的住宅 IP 登录、区域稳定，画像就健康；如果今天三个国家的机房 IP 轮着登，画像立刻就脏了。

这也是为什么“多人拼车”是头号高危——不是拼车这个行为本身违规，而是它必然导致**同一个号被一堆乱七八糟的 IP 登录**，画像想干净都难。低价区号被杀也是同理：注册时是 A 区的 IP，平时你用 B 区甚至机房 IP 登，区域常年对不上，风控不盯你盯谁。

所以与其纠结“我是不是说错了话”“是不是号本身有问题”，不如先把你能控制的那部分——**登录 IP——弄干净、弄稳定**。这是性价比最高、也最该先做的一步。

## 怎么自查：先看清楚你现在是从什么 IP 登的

动账号之前，先花一分钟看清楚自己的出口。打开 [ChatGPT · Codex IP 检测](https://ip.net.coffee/gpt/)，重点盯这几项：

| 看什么 | 健康的样子 |
| --- | --- |
| ChatGPT 出口 IP 属性 | 显示“住宅”才稳；显示“机房 / 数据中心”就是高危信号，趁早换。 |
| 信任评分 Trust Score | 70 以上比较安全；50 以下基本天天验证，离封号也不远。 |
| 出口 IP 区域 | 最好和你账号的注册区域对得上；至少别今天美国明天日本地乱跳。 |
| VPN / 代理 / Tor 标记 | 任意一项被标红，风控概率都会上去。 |
| 支持地区 | 出口落在 OpenAI 不支持的地区（中国大陆、香港等），再干净也用不了。 |

## 怎么防：六条能落地的

1. **一个号配一个固定的干净住宅 IP，别拼车、别多人共用。**这一条的优先级高于其他所有。省那点拼车钱，赔进去一个 Pro 号不值。
2. **账号注册区域，和你常用的登录 IP 区域对齐。**土区号就老老实实用土区附近的干净住宅出口，别拿个日本机房节点天天登土区号。
3. **别用机房 IP 登录。**大路货机场节点八成是机房出口，先用上面的工具确认 IP 属性是“住宅”再用。
4. **固定节点，别让出口乱跳。**关掉机场的“自动选线 / 负载均衡”，锁一个稳定线路，避免几小时横跨几国。
5. **关掉 IPv6。**双栈环境下 IPv6 容易绕过分流直连，造成出口分裂、区域漂移，间接把画像搞脏。细节看[为什么不建议用 IPv6 访问 ChatGPT / Codex](https://ip.net.coffee/gpt/ipv6.html)。
6. **Codex 重度用户悠着点。**别用非官方套壳工具驱动账号，别短时间怼出异常高的消耗，给账号留点“正常人”的样子。

> 一句话版本：**一号一个干净住宅 IP、区域对得上、别拼车、别机房、别乱跳。**这几条做到，你就避开了这轮 90% 的雷。

## 已经被封了还有救吗？

客观说，这轮封号申诉成功率不高。社区里多数人 `help.openai.com` 提了工单，等了很久没等到人工回复。能做的几件事：

- 正常渠道提申诉，把账号信息、订阅凭证写清楚，态度平和，能不能解封看运气。
- 如果是正价订阅、刚扣过费，且确实没违规，有人会走信用卡 chargeback（拒付争议）追回扣款——但这会让账号彻底凉，属于不想要了才用的下策，自行权衡。
- 别急着用同一个 IP、同一张卡再注册新号，否则新号大概率秒关联。先把 IP 和支付方式换干净。

说到底，封号这事预防远比补救划算。一个号能不能长命，很大程度上在你第一次登它的时候、用的是什么 IP，就已经决定了一半。

> 现在就去 [ChatGPT · Codex IP 检测](https://ip.net.coffee/gpt/)看一眼：你登 ChatGPT 用的出口 IP，是住宅还是机房？Trust Score 多少？区域对不对得上？一分钟，就知道自己的号在不在这轮的雷区里。想顺手查清楚 DNS 有没有漏，再过一遍 [DNS 泄露检测](https://ip.net.coffee/dns/)。
