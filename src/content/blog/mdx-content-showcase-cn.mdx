## 这篇文章是什么

这是一篇**可以直接构建**的 MDX 示例，而不是截图或伪代码。它把普通 Markdown 与项目内的 Astro 组件放在同一篇文章中：正文继续适合写作，重复出现的信息结构交给组件统一呈现。

> [!TIP]
> 普通文章优先写 `.md`。只有需要使用下方这类组件、原生 HTML 或可计算的数据时，再选 `.mdx`。

> 文章页会自动处理阅读时长、目录、标签、相关推荐、中英切换、浏览量和评论；作者不需要在正文里重复写它们。

## 普通 Markdown 仍然是主角

你可以使用**加粗**、*强调*、`行内代码`、[外部链接](https://astro.build/)、引用、分割线和普通列表：

- 一个要点
- 第二个要点
  1. 也可以继续分级
  2. 适合步骤或条件

- [x] 标题和摘要写清楚
- [x] 代码块标注语言
- [ ] 发布前在真实浏览器看一遍

## 这还不包含什么

这篇展厅故意没有伪装成“什么都支持”。当前未接通 Mermaid、KaTeX / MathJax、视频播放器、PDF / Bilibili 嵌入和 ECharts / Chart.js。

其中任何一个真的成为写作需求时，先新增一个集中维护、可访问并经过浏览器验收的组件；以后文章只导入它。这样不会让每篇文章都带着不同的临时实现。
