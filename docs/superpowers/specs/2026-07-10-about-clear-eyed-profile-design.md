# About「清醒档案」设计规格

- 日期：2026-07-10
- 状态：叙事方向已确认，等待书面规格复核
- 范围：重写并重新编排 About；新增一个可复用双角色对话卡；不修改首页、文章正文、后端或生产环境

## 目标

让 About 如实介绍站长：一名对 AI Agent、自动化和人机协作保持好奇的编程与开发爱好者，做过一些小工具、实验项目和个人工作流，但不自称资深工程师、专家、独立开发者、研究者或产品创始人，也不把尚未出现的代表作、成熟产品、用户规模和行业影响写成事实。

桑多涅作为本站虚拟助手与第三方观察者，负责整理、评价和引导；站长使用第一人称解释真实经历。页面吸引力来自诚实、好奇、愿意动手和愿意记录，而不是包装成就。

## 当前页面需要修正的问题

现有 About 中以下表达容易形成过度成熟的工程师叙事，应删除或降级：

- “个人技术索引”“值得反复验证的工程判断”容易暗示已形成稳定专业体系。
- “可部署、可回滚、可测试、可记录的系统”不能概括所有现有小项目。
- “工程偏好很明确”“保留证据链”可以写成个人习惯，不能写成已被证明的专业能力。
- “适合长期维护的个人技术档案”可以改为愿望或当前用途，不能暗示已持续维护多年。
- 项目描述不得使用“产品矩阵”“技术生态”“成熟系统”“代表作”等无事实支持的总结。

## 总纲

> 站长不是等待被发现的天才，也不是故意谦虚的成功者；他只是对 AI Agent 保持好奇，并愿意把一个个想法动手做出来的人。

双角色关系：

> 桑多涅不会替站长虚构成就。她可以指出他的普通、调侃他的不成熟，也会在确有依据时，承认那些微小但真实的进步。

### 3. 这些项目是什么

引导：

> 它们不是产品矩阵，也没有组成什么技术生态。大多数项目规模很小，通常从我自己遇到的问题，或者一个突然冒出来的想法开始。

## 可复用组件

新增 `src/components/CharacterDialogue.astro`，职责仅为呈现短对话或角色备注：

- `speaker`: 显示名称。
- `role`: 可选的小号身份说明。
- `avatar`: 可选头像；桑多涅默认使用 `/image/sandrone/dialog-chibi-v1.webp`。
- `variant`: `narrator` 或 `owner`，只控制边框、背景和说话者视觉差异。
- 默认插槽：短文案正文。

组件必须使用语义化 `aside` 或 `figure`，图片提供明确替代文本；移动端头像不遮挡正文。About 使用该组件，文章以后可以直接复用，但本次不批量修改文章。

## 视觉规则

- 保留现有站点字体、圆角、黑金基础和浅/深色主题，不另建一套《原神》UI。
- About 引导卡加入深红、旧金和冰蓝点缀，但角色主题不能压过正文。
- 桑多涅对话卡使用 Q 版透明素材作为角落装饰；站长回复使用无角色图的简洁卡片。
- 每个章节最多一次简短互动；长段事实使用普通正文，不写成剧本。
- 竖版观察者图不得在桌面或移动端裁掉脸、头饰、双手和主要服装结构。
- `prefers-reduced-motion` 下不新增必要动画；页面不依赖动画才能理解。

## 事实与语言验收

- 全文不得出现资深程序员、软件工程师、技术专家、独立开发者、创业者、创始人、AI 研究者、架构师或 Agent 专家等头衔。
- 不得声称成熟产品、长期大型项目、大量用户、商业成功、行业影响、技术生态或已经实现的长期价值。
- 桑多涅只肯定兴趣、行动、验证、记录、调整和微小进步。
- 站长真实经历与观点全部使用第一人称；桑多涅使用第三人称称呼站长。
- 鼓励面向过程，不出现成功必然、改变世界、技术前沿或伟大未来等口号。
- 调侃不羞辱、不连续自嘲，之后应有事实性的克制认可。
