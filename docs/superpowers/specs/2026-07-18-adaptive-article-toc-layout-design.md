# 文章目录与居中布局自适应设计

日期：2026-07-18

## 背景

文章页当前始终建立带左侧目录栏的网格。文章没有可识别标题时，目录组件虽然不显示内容，布局仍保留空侧栏占位，导致正文视觉上偏右。缺少章节导航因此首先表现为版式异常，而不是一个清晰的无目录状态。

写作流程也不应机械要求每篇文章达到固定章节数量。长文需要目录，短文和连续叙事则可能不适合为了目录硬拆标题。

## 目标

- 根据文章真实的 Markdown 标题自动选择目录布局或居中布局。
- 有导航价值的文章继续使用桌面目录与移动端目录抽屉。
- 不需要章节导航的文章不保留空目录栏，正文稳定居中。
- 审核稿使用原生 Markdown 表达章节，不引入专用组件或额外前端配置。
- 首次实现完成完整验证；后续普通文章继续使用简洁的 `ContentOnly` 通道。

## 非目标

- 不增加 `toc: true/false/auto` frontmatter。
- 不增加隐藏标题、自定义 Markdown 指令或 TOC 专用正文语法。
- 不改变目录的视觉样式、激活状态或滚动行为。
- 不要求每篇文章达到统一章节数量。
- 不让后续纯文章发布运行前端完整测试或浏览器矩阵。

## 方案

### 导航判定

文章渲染后继续从 Astro 的 `headings` 中提取深度为 2、3、4 的标题。至少有两个可用标题时，文章具备实际导航价值，启用目录布局；只有零个或一个可用标题时，使用无目录布局。

判定规则集中在 `BlogPost.astro`，桌面侧栏、移动端按钮和移动端抽屉共享同一个 `hasToc` 结果，避免各处阈值不一致。

### 布局行为

有目录时保留现有结构：

- 中等及以上视口显示左侧粘性目录；
- 移动端显示目录按钮与抽屉；
- 正文继续使用当前带目录的网格宽度。

无目录时：

- 不渲染桌面目录占位；
- 不渲染移动端目录抽屉，页头不显示目录按钮；
- 内容容器退出目录网格，正文以当前阅读宽度上限居中；
- 评论、相关文章、页脚和文章正文行为保持不变。

无目录不是异常或降级，而是短文与连续叙事的正式布局模式。

## 写作与审核约束

选题进入可审核稿时，必须明确选择以下一种内容结构：

1. **章节导航稿**：正文包含至少两个真实的 `##` 章节；需要细分时使用 `###`。标题可以写成自然过渡句，避免为了目录制造报告式小标题。
2. **短文居中稿**：正文不需要章节导航，可以不设置标题，或只有一个自然标题；页面自动使用居中布局。

审核时检查的是“结构选择是否符合文章”，而不是固定标题数量。不得加入空标题、无内容标题、HTML 隐藏标题或只为凑目录存在的章节。

结构选择直接由正文中的 Markdown 标题数量表达，不增加 frontmatter 字段或人工开关。

中英文配对文章保持相同的结构意图。允许译文根据语言习惯调整标题措辞，但章节对应关系不应无故分裂。

## 验证设计

### 首次实现

首次改动属于共享前端布局变化，需要执行：

- 针对 TOC 阈值和条件布局的自动回归测试；
- `npm run verify` 完整验证；
- 生产构建；
- 代表性有目录文章与无目录文章的桌面、移动端验收；
- 发布后的公开路由、控制台和资源请求检查。

测试至少证明：

- 两个以上标题启用目录布局；
- 零个或一个标题使用居中布局；
- 无目录状态不存在空侧栏占位；
- 移动端目录入口与桌面目录使用相同判定。

### 后续文章

首次实现上线后，普通 `.md` 文章仍按内容快速通道处理：

- 标题由现有 Markdown 渲染管线生成，不需要前端代码改动；
- 文章发布只运行内容配对、构建、资源与路由检查；
- 不因文章选择有目录或居中布局而升级为 `FastFrontend` 或 `FullAudit`；
- 只有修改布局、目录组件、样式或判定规则时才重新执行前端完整验证。

## 失败与回退

- 如果标题提取结果为空或不足两个，安全回退到居中布局，不显示空目录。
- 如果首次实现造成布局回归，使用前端版本化 release 回滚。
- 后续文章内容发布仍保留 `ContentOnly` 的版本化制品、哈希和原子切换能力。

## 完成标准

- 无标题文章在桌面和移动端正文居中，没有空目录栏。
- 只有一个标题的文章不显示单条目录。
- 至少两个可识别标题的文章保持现有目录体验。
- 项目规则和站点维护文档明确审核结构选择及后续快速发布边界。
- 首次完整验证与生产验收通过。
